# 하네스 엔지니어링 (LLM Agent Harness Engineering)

LLM 자체가 아니라 **LLM을 감싸는 스캐폴딩(하네스)** 을 설계하는 분야의 레퍼런스 모음입니다.
모델이 "두뇌"라면 하네스는 그 두뇌에 눈·손·기억·작업 절차를 붙여 실제로 일을 끝내게 만드는 시스템입니다.

> 핵심 명제: **에이전트의 품질 = 모델 능력 × 하네스 설계.**
> 같은 모델이라도 하네스(컨텍스트 구성, 도구, 루프, 검증)에 따라 결과가 크게 달라집니다.

## 왜 하네스인가

모델 단독 호출은 "한 번의 텍스트 생성"에 불과합니다. 실제 업무(코드 수정, 리서치, 운영)는
관측 → 판단 → 행동 → 검증을 여러 번 반복해야 끝납니다. 이 반복 루프와 그 안에서 모델에게
무엇을 보여주고(컨텍스트), 무엇을 할 수 있게 하고(도구), 언제 멈출지(종료 조건)를 결정하는 것이
하네스 엔지니어링입니다.

## 빠르게 보는 설계 체크리스트

- [ ] **컨텍스트**: 매 턴 모델에 들어가는 토큰이 의도적으로 예산화되어 있는가? (불필요한 덤프 제거)
- [ ] **도구**: 각 도구가 단일 책임 + 명확한 스키마 + 회복 가능한 에러 메시지를 갖는가?
- [ ] **루프**: 종료 조건이 명시적인가? (무한 루프 / 조기 종료 방지)
- [ ] **검증**: 행동 결과를 모델이 관측하고 스스로 교정하는 단계가 있는가?
- [ ] **메모리**: 세션을 넘겨 보존할 사실과 버릴 사실이 구분되는가?
- [ ] **안전**: 되돌리기 어렵거나 외부로 나가는 행동에 확인/권한 게이트가 있는가?
- [ ] **관측**: 실패를 재현·추적할 로그/트레이스가 남는가?
