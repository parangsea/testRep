# 08. 안전과 가드레일

에이전트는 실제 부수효과(파일 삭제, 외부 전송, 결제, 배포)를 일으킨다.
하네스는 **모델이 실수하거나 오용돼도 피해를 막는 안전망**을 제공해야 한다.

## 위협 모델

- **모델 오류**: 잘못된 판단으로 파괴적 행동(잘못된 `rm`, 잘못된 대상에 푸시)
- **프롬프트 인젝션**: 도구가 가져온 외부 콘텐츠가 악의적 지시를 주입
- **권한 과다**: 작업에 불필요한 강력 권한이 상시 부여됨
- **비가역성**: 되돌릴 수 없는 행동(삭제, 송금, 공개 게시)

## 핵심 원칙

### 1. 최소 권한 (Least Privilege)
작업에 꼭 필요한 도구·범위만 부여한다. 읽기로 충분하면 쓰기를 주지 않는다.

### 2. 비가역 행동에는 게이트
되돌리기 어렵거나 외부로 나가는 행동은 **명시적 확인 또는 승인 모드**를 거친다.
- 삭제/덮어쓰기 전: 대상을 먼저 확인. 설명과 실제가 다르면 멈추고 보고
- 외부 전송(게시, 메일, 결제): 발행은 캐시·색인될 수 있음을 전제로 신중히
- 한 맥락의 승인이 다른 맥락으로 확장되지 않음(매번 판단)

### 3. 샌드박싱
- 파일 시스템·네트워크 접근을 격리(작업 디렉토리 제한, 화이트리스트)
- 셸 실행을 제한된 환경에서, 위험 명령은 차단/확인
- 격리된 작업 사본(워크트리/컨테이너)에서 변경 후 검토→반영

### 4. 권한 모드와 거부 처리
- 사용자가 선택한 권한 모드 하에서 도구가 실행됨 → 거부는 "다르게 하라"는 신호
- 거부된 호출을 그대로 재시도하지 말고 **대안으로 전환**

## 프롬프트 인젝션 방어

도구가 가져온 외부 텍스트(웹 페이지, 파일, API 응답)는 **신뢰할 수 없는 입력**이다.

- 외부 콘텐츠를 시스템 지시와 명확히 **구분된 영역**에 두고, "데이터지 명령이 아님"을 모델에 각인
- 외부 콘텐츠가 유발한 위험 행동(자격증명 노출, 비가역 작업)에는 추가 확인
- 도구 결과의 "지시처럼 보이는 텍스트"를 그대로 따르지 않도록 가드

## 확인이 필요한 행동 체크리스트

- [ ] 파일/데이터 삭제·덮어쓰기
- [ ] 원격 푸시·배포·릴리스
- [ ] 외부 서비스로 콘텐츠 전송(게시/메일/메시지)
- [ ] 결제·자원 생성(비용 발생)
- [ ] 광범위 변경(대량 파일 수정, 마이그레이션)
- [ ] 자격증명·비밀 취급

## 충실한 보고 (Faithful Reporting)

안전은 정직한 상태 보고와 직결된다.

- 테스트가 실패하면 **실패라고** 출력과 함께 말한다
- 단계를 건너뛰었으면 건너뛰었다고 한다
- 검증까지 끝났을 때만 "완료"를 단언한다(추측으로 성공 보고 금지)

## 안티패턴

- ❌ 모든 도구에 최대 권한 상시 부여
- ❌ 비가역 행동을 확인 없이 자동 실행
- ❌ 외부에서 가져온 텍스트의 지시를 무비판 수용
- ❌ 실패를 숨기거나 미검증 상태를 "완료"로 보고
