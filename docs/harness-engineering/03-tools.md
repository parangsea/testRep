# 03. 도구 설계

도구(tool/function)는 모델이 외부 세계에 영향을 주는 **유일한 통로**입니다.
도구 설계 품질이 에이전트가 할 수 있는 일의 범위와 신뢰성을 결정합니다.

## 좋은 도구의 원칙

1. **단일 책임** — 하나의 도구는 하나의 명확한 일을 한다. "만능 도구"는 모델이 오용한다.
2. **명확한 스키마** — 파라미터 이름·타입·설명이 자기설명적이어야 한다. 모델은 스키마만 보고 호출한다.
3. **회복 가능한 에러** — 실패 메시지는 모델이 **다음 행동을 교정**할 수 있게 작성한다.
4. **결정적이고 안전한 기본값** — 위험한 동작은 명시적 옵트인, 안전한 쪽이 기본.
5. **적은 개수** — 도구가 많을수록 선택 오류가 늘고 토큰을 먹는다. 꼭 필요한 것만.

## 스키마 설계

```jsonc
{
  "name": "read_file",
  "description": "파일을 읽어 내용을 반환한다. 큰 파일은 offset/limit로 일부만 읽어라.",
  "parameters": {
    "file_path": { "type": "string", "description": "절대 경로" },
    "offset":    { "type": "number", "description": "시작 라인(선택)" },
    "limit":     { "type": "number", "description": "읽을 라인 수(선택)" }
  }
}
```

- **설명에 사용법·제약을 담아라**: "절대 경로", "큰 파일은 부분 읽기" 같은 힌트가 오용을 줄인다.
- **이름은 동사+명사**: `read_file`, `search_code`, `run_tests` — 의도가 드러나게.
- **열거형(enum)으로 선택지를 좁혀라**: 자유 문자열보다 모델 실수가 적다.

## 도구 결과 포매팅

모델이 **읽을** 출력임을 항상 기억하라. 사람용 로그가 아니다.

- 일관된 구조(헤더, 구분자, 메타데이터)
- 핵심 신호를 앞에, 잡음을 제거 — 1만 줄 로그를 그대로 반환하지 말 것
- 위치 정보(파일 경로 + 라인)를 붙여 후속 행동의 근거 제공
- 너무 길면 잘라내되 **"잘렸음"을 명시** (조용한 절단은 "다 봤다"는 착각을 부른다)

```
# 좋은 예: 검색 결과
3건 일치 (총 120줄 중 상위 3건):
  src/api/client.ts:6   baseURL 설정
  src/config.ts:2       API_BASE_URL 정의
  ...
```

## 에러 처리

에러는 **모델에게 보내는 메시지**다. 단순 실패가 아니라 교정 신호로 설계한다.

| 나쁜 에러 | 좋은 에러 |
| --- | --- |
| `Error: ENOENT` | `파일을 찾을 수 없음: /x/y.ts. 경로를 확인하거나 먼저 디렉토리를 나열하라.` |
| `Invalid input` | `offset은 1 이상이어야 함(받은 값: -3).` |
| (조용히 빈 결과) | `일치 항목 없음. 패턴을 넓히거나 대소문자 무시를 시도하라.` |

## 병렬·지연 도구

- **병렬 실행**: 서로 의존하지 않는 도구 호출은 한 번에 내보내 지연을 줄인다. 하네스가 동시 실행을 지원해야 한다.
- **지연 로딩(deferred tools)**: 도구가 매우 많으면 전체 스키마가 컨텍스트를 잠식한다. 이름만 노출하고 **필요 시 스키마를 검색/로드**해 토큰을 절약한다.
- **의존 호출은 순차**: 앞 결과가 뒤 입력에 필요하면 병렬화하지 말 것.

## 권한·부수효과

- 읽기 전용 도구와 **부수효과(쓰기/삭제/네트워크) 도구를 구분**하라.
- 되돌리기 어려운 행동은 확인 게이트 또는 권한 모드로 보호 → [08. 안전과 가드레일](./08-safety-and-guardrails.md)
- 도구가 거부됐을 때(권한 미승인) 모델이 우회/대안으로 전환하도록 결과에 그 의미를 담아라.

## 안티패턴

- ❌ 파라미터 설명 없는 모호한 스키마
- ❌ 사람용 장황한 로그를 그대로 결과로 반환
- ❌ 실패를 빈 문자열/널로만 알림(교정 정보 없음)
- ❌ 한 도구에 너무 많은 모드를 몰아넣어 분기 폭발
